Content standard 3.0: Understanding Web Page Design and Layout


Standard 3 applies to Forms and the two kinds of Tables. Layout and Data tables are addressed in standard 3.1, while forms are addressed in 3.2. No current examples for 3.1 exists except for a detailed explanation. 3.2 can be demostrated by a form created for the VRBO website project, which contains a contact form.


standard 3.1: Appropriately Use Tables to Enhance their Web Pages

Standard 3.1 covers the difference of table layouts and data tables. A data table is as simple as the name sugests, a table in which data is stored. A layout table is a primitive form of styling that was used to order and "style" html documents without using CSS, and creates a crude layout taht can be quickly accomplished with CSS. I rarley use tables, but positioning with td and tr tags are fairly simple(td is listed under tr, and these tags can be modified with CSS).


standard 3.2: Demonstrate the Ability to Use Design and Layout Web Forms

Forms are an essentail part to websites because they allow interaction between the user and the site. Whether this be by text fileds, buttons, drop down selection fields, or check boxes. Forms can provide a form of server communication to provide, feedback, make requests, or so on so forth. In my VRBO site, the contact aside bar provides a vary basic form with a PHP link to email. They also include form validation, which has required boxed taht need to be filled in order submit the field.
